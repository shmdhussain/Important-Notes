Here are the Important points on iCinema:
1.	Your HTML files should contain only required tags, i.e., display elements, layout info and static content. Style should be in CSS and event handlers should be in .js files
2.	For OOJS design, split the data in supplied JSON to corresponding JS classes as discussed. Users.js, Movies.js, Screens.js, BookingDetails.js and the wrapper for all iCinemaWrapper.js should all be present.
3.	Add all the business logic code to iCinemaWrapper.js as methods and all the global variables and of course, user, movie etc object as properties in it.
4.	In the EventHandler.js, have the $(document).ready();  in which you are creating the JSON objects, creating Users, Movies, Screens, Transactions and Wrapper objects. Use the data from JSON to populate the corresponding objects
5.	Then, you can call the event handlers  eg: click and hover event functions in there itself, which would be calling the corresponding method of the Wrapper object
6.	Use classes in place of element or id in css.

iCinemas, a premier multiplex in the district wants to allow public to books tickets for its shows online.

They have contacted you to design and create the website for them.

The set of requirements they have given you are:

#In the home page
-----------------
	-show posters of the movie being shown and/or coming soon, to aid the user to make the choice of movie. (Feel free to use your creativity here)
	-allow the user to signup using phone no, email and a password of user's choice. Once registered, the user can login using emailID and accepted password.
	-once the user logs in, display appropriate welcome message eg: Welcome, emailID
	-allow the user to enter/select the date (only till the immediate next Friday). Polulate the list of movies based on the date (This actually does not need the user to have logged in)
	-once the user selects the movie, navigate to show times section (either a new page or same page). Show times for any movie are 10:30 1:30, 4:30, 7:30 and 9:30, per screen

#Once the user selects a show
-----------------------------

	-show the user the seating arrangement of the screen pictorially (assume 100 seats per screen,numbered from 1-100, 6 screens)
	-allow the user to select/enter the no: of seats (max allowed 10).
	-once the seat no: have been selected, user can select the seats in the graphical representation of the seating arrangement from the available seats.
	  eg: if the user wants 4 seats, the seat he clicks plus 3 seats to the right of it should get selected and appear in a different color.
	-once the user has selected the seats and proceeds to booking, ask the user to login/register if s/he has not done so.
	-display the amount (250 for 2D and 350 for 3D) to the user with the message "Ticket Reservation Successful. Confirmation SMS has been sent to your registered number. Kindly collect the tickets 15 prior to the show time by paying at the counter". On Ok, take the user back to home page.

#For a user
-------------
User's transactions should be tracked. A user once he has logged in, should be able to see past bookings with movie name, date and no of tickets.

Provided Data:
---------------
6 screens
100 seats per screen, numbered from 1 to 100
2D movie rate 250
3D movie rate 350

We have based these set of requirements on BookMyShow.com. So, for your user research think like the end user and to get the insights you can refer that site.

We have reduced the functionalities so that we can focus on design principles and User Experience.
